# Bug Fixing Discipline (必须遵守)

## 🔴 第一步强制(每个 bug 修复前必做)

**调 Skill 工具运行 `superpowers:systematic-debugging`** —— 不靠直觉,先走系统化流程。

- 不到 1 分钟就能省掉后续 30 分钟的绕圈
- 历史数据(2026-06-07 审计):该 skill 仅被用 18 次/111 会话(0.16/会话),`test-driven-development` 仅 20 次(0.18/会话)—— 实施层纪律严重薄弱
- 累积后果:项目 15+ pre-existing 测试失败(mocks.test.js 13 + dictation_bugs.test.js 2)

---

## 规则

1. **修复 bug 必须按顺序**:先复现 → 贴报错 → 定位根因 → 最小修复 → 验证通过,再标记 done。
2. **禁止无验证的"已修复"**;禁止用静态分析替代真实执行。
3. **一次只改一个问题**,不批量修复;改完必须跑对应测试/复现步骤。
4. **若同一问题修复 2 次未解决,停止修复,重新分析根因**,不许绕圈。
